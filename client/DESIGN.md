# Irodori Chat — デザインガイドライン

静かで親密な会話のための設計言語。

LAN 内で動くキャラクター会話アプリ「Irodori Chat」のクライアントアプリ UI のデザインシステム。会話に集中でき、長時間使っても疲れない——温かみのある白を基調に、近黒のインクとやわらかい影だけで構成する。色は状態表示にのみ最小限に用いる。

**原則**: 静か / 親密 / 会話が主役 / やわらかな白＋ベージュの差し色 / 装飾は最小

すべてのトークンは [src/styles.css](src/styles.css) の `@layer tokens` に CSS カスタムプロパティとして定義されている。本ドキュメントはその意図と運用ルールを示す。

---

## 01 カラーパレット

色相を持たないニュートラルが土台。サーフェスはわずかに暖色へ寄せた「紙」のような白、テキストは黒ではなく暖かい近黒。彩度のある色は接続状態の 3 トークンのみに限定し、画面の主役にしない。全色 `oklch()` で定義し、明度のステップを揃える。

### サーフェス / 背景

| トークン      | 値                      | 用途                     |
| ------------- | ----------------------- | ------------------------ |
| `--bg`        | `oklch(0.968 0.004 95)` | アプリ全体の背景         |
| `--bg-tint`   | `oklch(0.945 0.005 95)` | 背景グラデの陰側         |
| `--surface`   | `oklch(1 0 0)`          | カード・吹き出し・入力   |
| `--surface-2` | `oklch(0.985 0.004 95)` | 入力欄・パネル下地       |
| `--surface-3` | `oklch(0.965 0.004 95)` | セグメント・キーキャップ |

### インク / テキスト

| トークン      | 値                      | 用途                     |
| ------------- | ----------------------- | ------------------------ |
| `--ink`       | `oklch(0.235 0.008 70)` | 本文・見出し・主ボタン地 |
| `--ink-soft`  | `oklch(0.46 0.008 70)`  | 補助テキスト・ラベル     |
| `--ink-faint` | `oklch(0.63 0.006 70)`  | キャプション・メタ       |
| `--ink-ghost` | `oklch(0.74 0.004 70)`  | プレースホルダ・最弱     |

### ライン / 境界

| トークン        | 値                      | 用途                   |
| --------------- | ----------------------- | ---------------------- |
| `--line`        | `oklch(0.915 0.004 90)` | 区切り線・標準ボーダー |
| `--line-strong` | `oklch(0.86 0.004 90)`  | 入力枠・強い境界       |

### アクセント — 主アクションのベージュ

主アクション・選択状態の塗りに使うやわらかいベージュ。黒(`--ink`)の強い塗りに代えて、温かく穏やかな印象を与える。低彩度の暖色なので、画面の無彩色を壊さず差し色として馴染む。

| トークン          | 値                     | 用途                                           |
| ----------------- | ---------------------- | ---------------------------------------------- |
| `--accent`        | `oklch(0.85 0.045 78)` | 主ボタン・送信・選択チップ・音声再生ボタンの地 |
| `--accent-strong` | `oklch(0.8 0.055 78)`  | hover・押下・トグル on の地                    |
| `--accent-ink`    | `oklch(0.33 0.035 65)` | ベージュ地の上のテキスト/アイコン(濃い暖色)    |

**運用ルール** — ベージュ地の上の文字・アイコンは白ではなく `--accent-ink`(濃い暖色)を使う。利用者の吹き出しは話者識別のため `--ink`(黒)の塗りを維持し、ベージュには置き換えない。

### 状態アクセント

| トークン | 値                     | 用途                |
| -------- | ---------------------- | ------------------- |
| `--ok`   | `oklch(0.66 0.13 150)` | 接続済み / 正常     |
| `--warn` | `oklch(0.78 0.12 80)`  | 接続待ち / 注意     |
| `--err`  | `oklch(0.62 0.17 25)`  | エラー / 破壊的操作 |

**運用ルール** — 状態色は「7px のドット」(`.dot`) または小バッジに限定する。背景・大面積・ボタン地には使わない。破壊的操作(履歴クリア等)のみ `--err` をテキスト/枠に薄く用いる(`.btn.danger`、`.alertline`)。

## 02 タイポグラフィ

和文に最適化した **Zen Kaku Gothic New** 一書体で全体を統一。静かで素直な字面が「親密さ」を支える。ウェイトは 300 / 400 / 500 / 700 の 4 段階のみ。本文の行間は広め(1.7前後)で、長い会話でも読み疲れしない。数値・URL・モデル名・時刻は等幅にする。

| トークン | 値                                                                  | 用途                |
| -------- | ------------------------------------------------------------------- | ------------------- |
| `--font` | `"Zen Kaku Gothic New", "Hiragino Sans", "Noto Sans JP", system-ui` | 既定書体            |
| `--mono` | `ui-monospace, "SFMono-Regular", Menlo`                             | 数値・URL・モデル名 |

フォントは Google Fonts から読み込む([index.html](index.html))。LAN-only のオフライン環境では Hiragino Sans / Noto Sans JP へ自然にフォールバックする。

### タイプスケール

ユーザーのブラウザ設定を尊重するため `rem` で定義する(括弧内は 16px 基準の px 相当)。

| トークン       | 値               | 用途                             |
| -------------- | ---------------- | -------------------------------- |
| `--fs-micro`   | 0.6875rem (11px) | ヒント・最弱メタ・パネル小見出し |
| `--fs-caption` | 0.75rem (12px)   | 補足・ラベル・話者名・メタ       |
| `--fs-ui`      | 0.8125rem (13px) | ボタン・入力・UI                 |
| `--fs-ui-lg`   | 0.875rem (14px)  | 小見出し・会話ヘッダーのタイトル |
| `--fs-body`    | 0.9375rem (15px) | 本文・吹き出し                   |
| `--fs-name`    | 1.375rem (22px)  | キャラクター名                   |

- 行間: 本文 1.72 / 見出し 1.1〜1.3 / UI 1.5
- パネルのセクション見出し(`.sect > .h`)は 11px・weight 700・`letter-spacing: 0.12em`・UPPERCASE
- 吹き出しは `text-wrap: pretty` + `overflow-wrap: anywhere` で、長い URL でもはみ出さない

## 03 余白

4px を基本単位とした段階的スケール。余白は「詰める」より「ゆとり」を優先し、要素同士は margin ではなく **flex / grid + `gap`** で明示的に離す(並べ替え・削除に強い)。

| 段階 | px   | 用途の目安               |
| ---- | ---- | ------------------------ |
| 1    | 4px  | アイコンと文字の最小間隔 |
| 2    | 8px  | チップ内・隣接要素       |
| 3    | 12px | 入力欄パディング・行間   |
| 4    | 16px | フィールド間・カード内   |
| 5    | 20px | パネル左右・セクション内 |
| 6    | 26px | カードパディング・レール |
| 8    | 40px | 会話エリア左右(広幅)     |

画面外周や本文幅は `clamp(min, vw, max)` で可変にする(例: スレッドの左右余白 `clamp(18px, 5vw, 64px)`)。

## 04 コンポーネント基本ルール

部品は「白いサーフェス＋細いライン＋やわらかい影」で統一。主アクションの強調は **やわらかいベージュの塗り(`--accent`)** で行い、その上の文字・アイコンは `--accent-ink`(濃い暖色)にする。角はやわらかく、押下時はわずかに沈む。タップ対象は最低 44px を確保する。

### ボタン (`.btn`)

| バリアント         | 地 / 枠                                           | 用途                            |
| ------------------ | ------------------------------------------------- | ------------------------------- |
| `.primary`         | `--accent` 地 / `--accent-ink` 文字 / `--sh-2`    | 主アクション(保存)。画面に 1 つ |
| (default)          | `--surface` 地 / `--line` 枠 / `--sh-1`           | 標準操作(設定を開く・接続等)    |
| `.ghost`           | 透明 / 枠なし                                     | 補助・取り消し                  |
| `.danger`          | `--err` 文字 / 薄い赤枠                           | 破壊的操作のみ                  |
| `.send` (円形)     | 44px 円 / `--accent` 地 / `--accent-ink` アイコン | 送信。disabled は `--surface-3` |
| `.icon-btn` (円形) | 40px 円 / `--line` 枠                             | アイコン操作(設定ギア・マイク)  |

hover で `translate: 0 -1px` ＋影を一段上げ、active で元に戻す。disabled は地を `--surface-3`・文字を `--ink-ghost` にして影を消す。

### チップ (`.chip`)

選択トグル(口調プリセット等)。選択状態 `.on` は `--accent` 塗り＋`--accent-ink` 文字。未選択は `--surface-2` 地＋`--line-strong` 枠。`aria-pressed` を付ける。

### 入力欄 (`.input` / `.select` / `.area`)

地 `--surface-2`・枠 `--line-strong`・`--sh-inset` でわずかに凹ませる。フォーカスで枠を `--ink-soft` にし、半透明の黒リングを足す。角丸 `--r-md`。

### コンポーザー (`.composer .field`)

複数行入力は `--r-xl` のピル型。`field-sizing: content` で入力に応じて自動伸長(最大 140px。非対応ブラウザは JS フォールバック)。フォーカスは `:focus-within` で枠色と影を一段上げる。

### カード (`.preview-card` / `.status-item`)

白サーフェス＋`--line` 枠＋`--sh-2`。角丸 `--r-lg`。装飾カードの多用は避け、情報のまとまりが必要な箇所のみに使う。

### 会話の吹き出し (`.msg` / `.bubble`)

- **利用者** = `--ink` 塗りの右寄せ。右下角を 6px に詰める
- **キャラクター** = 白カード(`--line` 枠)の左寄せ。左下角を 6px に詰める
- 話者の区別を色ではなく **塗り/枠と左右位置** で表す
- 新着は下から 6px のフェードイン(`.fresh`、`prefers-reduced-motion` 時は無効)
- 最大幅 720px(モバイルは 86%)

### 音声チップ (`.audiochip`)

ブラウザ標準の `<audio controls>` は使わず、カスタムプレイヤーで統一する。30px の `--accent` 円の再生ボタン(アイコンは `--accent-ink`)＋細い波形(再生中のみアニメーション)＋等幅の時間表示。実体の `<audio>` 要素は非表示で保持する。

### 返答中 (`.typing`)

キャラクター側の吹き出しと同じ器に、点滅する 3 ドット＋「◯◯が返答中…」。装飾的な無限ループアニメは、この状態表示(点滅ドット・波形)に限る。

## 05 シャドウ ・ 角丸 ・ ボーダー

影はすべて黒のみ(色を持たせない)。近距離の鋭い影と遠距離の広い影を重ねた多層構成で、紙が浮くようなやわらかさを出す。彩度のあるグロー・ネオンは使わない。

### Shadow トークン

| トークン     | 用途                                   |
| ------------ | -------------------------------------- |
| `--sh-1`     | 標準ボタン・ステータス・最小の浮き     |
| `--sh-2`     | カード・吹き出し・主ボタン・入力       |
| `--sh-3`     | フォーカス時のコンポーザー・hover 強調 |
| `--sh-4`     | スライドオーバーパネル                 |
| `--sh-inset` | 入力欄の凹み(内側の影)                 |

### Radius トークン

| トークン   | 値    | 用途                                 |
| ---------- | ----- | ------------------------------------ |
| `--r-sm`   | 8px   | 小要素・キーキャップ                 |
| `--r-md`   | 12px  | 標準ボタン・入力欄・ステータス項目   |
| `--r-lg`   | 18px  | カード・吹き出し・キャラクター画像   |
| `--r-xl`   | 26px  | コンポーザー(複数行入力のピル)       |
| `--r-pill` | 999px | チップ・ステータスドット・円形ボタン |

### Border

| 用途             | 指定                              |
| ---------------- | --------------------------------- |
| 区切り線・標準枠 | `1px solid var(--line)`           |
| 入力欄・強い境界 | `1px solid var(--line-strong)`    |
| ドロップゾーン   | `1.5px dashed var(--line-strong)` |

### モーション

- 全トランジションは `--ease: cubic-bezier(0.22, 0.61, 0.36, 1)`、時間は 0.16〜0.34s
- アニメは `opacity` / `transform` / `translate` を優先(コンポジタで動く)
- `prefers-reduced-motion: reduce` では、スライド・バウンス・パルスを止めるか不透明度のみの控えめな表現に置き換える

## 06 レスポンシブの考え方

「キャラクターの存在」と「会話」を、広い画面では横に並べ、狭い画面では縦に積む。会話エリアは常に主役として最大化し、付帯情報(キャラクター画像・接続状態・設定)はモバイルでは折りたたんでヘッダー／スライドオーバーへ退避する。

| 項目                 | 方針                                                                                                                           |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| ブレークポイント     | `760px` を境に 2 カラム ⇄ 縦積みを切り替える                                                                                   |
| デスクトップ ≥ 760px | `grid-template-columns: 348px 1fr`。左にキャラクターレール(画像・名前・状態・キャラクター設定の要約・設定ボタン)、右に会話     |
| モバイル < 760px     | レールを隠し、小アバター＋名前＋状態＋設定ギアを会話ヘッダーに集約。パネルは全幅                                               |
| 高さ                 | アプリは画面高にフィット(`100dvh`)。ヘッダー・コンポーザーは固定高、**スレッドのみスクロール**(`overscroll-behavior: contain`) |
| 外周・本文幅         | `clamp(18px, 5vw, 64px)` で余白を可変に。`overflow-wrap: anywhere` ではみ出しを防ぐ                                            |
| ヒット領域           | タップ対象は最低 44px。送信・アイコンボタンは 40〜44px 円                                                                      |
| 将来                 | Tauri 化・スマホ実機を見据え、レイアウトはビューポート基準(固定 px の絶対配置に依存しない)                                     |

## 07 設定パネル(スライドオーバー)

設定は別画面ではなく、右からのスライドオーバーで開く。

- ネイティブ `<dialog>` + `showModal()` を使う(フォーカストラップ・Esc・背景の不活性化が標準で付く)
- ライトディスミスは `closedby="any"` を宣言し、非対応ブラウザ(Safari)には座標判定の JS フォールバックを併用する
- 開閉は `transform: translateX(100%)` のスライド。`@starting-style` + `transition-behavior: allow-discrete` で `display` を含めて遷移させる
- 背景は `::backdrop` を `rgb(20 18 15 / 28%)` へフェード
- 幅 408px(モバイルは全幅)。`--sh-4` で大きく浮かせる

セクション構成: キャラクター / 性格・口調 / 声・読み上げ / 接続 / 履歴。セクション小見出しは UPPERCASE のトラッキング広め。サーバーに保存される設定(キャラクター名・キャラクター設定・口調プリセット・距離感・読み上げ設定)は「保存する」で明示的に**設定保存**し、保存により会話履歴がクリアされることをヘルプテキストで伝える。MVPでは話者選択を表示せず、Irodori-TTS-Server側のno-ref音声設定を使う。

将来機能のプレビュー UI(口調プリセット・距離感・話す速さ)はローカル保存のみで、会話サーバーへは送られない。その旨をヘルプテキストで明示する。

## 08 アクセシビリティ

- フォーカスリングは `:focus-visible` で `outline: 2px solid var(--ink-soft)` + `outline-offset: 2px`。`outline: none` で消す場合は代替のフォーカス表現(コンポーザーの `:focus-within` 等)を必ず用意する
- 状態の変化(接続しました・返答中など)は `aria-live="polite"`、エラーは `role="alert"` で通知する
- トグルは `role="switch"` + `aria-checked`、選択チップは `aria-pressed`
- 色だけで意味を伝えない: 状態ドットには必ずテキストラベル(接続済み・要確認など)を添える
- 日本語 IME での確定 Enter は送信しない(`KeyboardEvent.isComposing` を確認)
