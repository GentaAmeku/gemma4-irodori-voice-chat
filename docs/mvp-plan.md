# MVP Plan

## Goal

最初のMVPでは、PCブラウザ上のSvelteクライアントからLAN内の会話サーバーへ接続し、テキスト入力からAI応答生成、読み上げ音声生成、画面表示、音声再生までを通す。

音声入力、Tauri化、スマホ実機対応は、この縦切りが動いてから追加する。

## MVP Scope

### Conversation Server

- FastAPIでREST APIを提供する。
- Ollamaとirodori-TTSは起動済み前提にする。
- 起動時またはhealth取得時にOllamaとirodori-TTSの疎通状態を返す。
- テキスト会話ターンを同期RESTで処理する。
- 同時に進行できる会話ターンは1つだけにする。
- busy時は後続入力を拒否する。
- キャラクター設定、口調プリセット、距離感、読み上げ設定、キャラクター画像をサーバー側で管理する。
- 設定保存時はメモリ上の会話履歴をクリアする。
- 会話履歴はMVPではメモリ保持にする。
- LLMにはキャラクター設定、口調プリセット、距離感、今回入力、直近10ターンを渡す。
- 生成音声は一時ファイルとして公開URLを返す。

### Client

- Svelte + TypeScript + Viteで実装する。
- 最初はWebクライアントとしてPCブラウザで動かす。
- 会話サーバーの接続先をUIで編集できる。
- 最後に成功した接続先をローカル保存する。
- healthを表示し、未接続、接続中、利用可能、会話中、エラーを区別する。
- テキスト入力で会話ターンを開始できる。
- 応答テキストと生成音声を受け取り、履歴表示と音声再生を行う。
- キャラクター名、キャラクター設定、口調プリセット、距離感、読み上げ設定をUIで編集して明示保存できる。
- 話者選択はMVP UIには出さず、`speaker_id: "none"` とIrodori-TTS-Server側のno-refカスタマイズで扱う。
- キャラクター画像をアップロードして差し替えできる。
- 履歴クリアを実行できる。

## Implementation Order

1. `server/` を作成し、FastAPIアプリの骨格を作る。
2. サーバー設定のload/save、履歴管理、busy制御を実装する。
3. Ollama / irodori-TTS連携をアダプタとして分離する。
4. テキスト会話REST APIを実装する。
5. サーバーの単体テストを追加する。
6. `client/` を作成し、Svelte + TypeScript + Viteの骨格を作る。
7. 接続先設定、health表示、テキスト会話UIを実装する。
8. 設定パネル、画像アップロード、履歴クリアを実装する。
9. サーバーとクライアントを起動し、少なくともモックまたは実サービス設定で縦切りを確認する。

## Done Criteria

- `server` のテストが通る。
- `client` のTypeScriptチェックまたはビルドが通る。
- PCブラウザで接続先を設定し、health状態を確認できる。
- テキスト入力から会話ターンを送信できる。
- 会話サーバーが応答テキストと音声URLを返す。
- クライアントが応答テキストを履歴表示し、音声URLを再生対象として扱える。
- 設定保存、履歴クリア、キャラクター画像アップロードのUIとAPIが存在する。

## Deferred

- 音声入力、VAD、STT、音声チャンクWebSocket送信。
- Tauri PCアプリ化。
- スマホ実機検証。
- 会話履歴の永続化。
- 読み上げON/OFF。
- 参照音声登録と話者選択UI。
- 共有トークン認証。
- LLM/TTSストリーミング。
- irodori-TTS詳細パラメータ編集。
- Playwright自動E2E。
